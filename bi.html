One of the challenges in modern BI is keeping up with the trends.

More than that, how can you UPDATE your business to keep up with the demands, while maintaining control?

It is nearly impossible to anticipate change, but you can always do your best to roll with the changes. One such concept is Data Source Agnosticism.

This is the practice of not requiring a specific technology platform, so if one fails, your business will not suffer total loss of functionality.

In cloud technologies, this is very important. Due to data import/export policies on cloud systems, it is prudent to have a mix, or hybrid approach.

This hybrid approach is best defined as focusing on multiple vendors and database types, and tools that let you bring it all together.

In one of my previous roles, I learned how to create and develop an on-premises Business Intelligence solution through Power BI Report Server.

There are a few things that set it apart from other tools:

  • No limits on data refreshes and storage, since you control the server. (Others: Tableau Server, SAS Enterprise BI)
  • It is founded on well-known technologies:
    • REST API
    • PowerShell
    • T-SQL
    • SSRS
    • C# ASP.NET
    • PowerQuery / M
    • DAX
  • On-Prem and Cloud BOTH. This matters because:
    • You may want cloud features (Machine Learning / Artificial Intelligence) and on-premises data control, depending on the report data in question.
    • Data Security - One of the most pressing features these days is cybersecurity. PBIRS is useful because it exists behind your firewall, and is largely controlled by your Organization.
    • Existing Legacy BI integration (SSRS, Access, Excel, Office Online) - This tool is one of the best at using other products in conjunction with itself.
    • 106 Data Sources - One of the best data consumers, with the right Gateway/oAuth Proxy set-up, this will let you use Cloud sources, or use an SSRS Dataset in a Power BI Report.
  • The most frustrating part is the licensing:
    • It comes *free* with an Enterprise SQL Server License, OR
    • You can opt for Power BI Premium which allows for increased Cloud refreshes/usage/functionality,
    • AND you must still have report creators with a Pro license, using either of the above options.
One of the things people don't realize, is that you can use this and other BI tools, and sometimes they overlap, such as both can use Analysis Services or SQL Server Tables. Typically, there are two things that affect adoption of the tool the most:
  • Cost
  • Polish
